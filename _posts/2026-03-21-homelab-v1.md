#### why homelabbing?
a homelab is a self hosted server environment used for testing, learning and experimenting with different projects or tools. a few months ago, i started working on a small home lab in my apartment. lurking in posts of [`/r/homelab`](https://www.reddit.com/r/homelab/) or [`/r/selfhosted`](https://www.reddit.com/r/selfhosted/) helped me shape this project overtime.<br>
a little bit background to why i started it, i own a personal vultr server where i host my personal scripts and crons to automate some of my tasks, and i was thinking of creating a home server to run these scripts and other open-source tools. i also have few subscriptions like netflix, prime, youtube music which could be replaced by a good home lab. also, as a full time developer, i think it would be a good side project for me to learn new things during the weekends.

##### [pi-hole](https://pi-hole.net/)
for the first homelab application, i deployed a docker continer of pi-hole. a few years ago, during my high school, we configured a pi hole server in my home network back in nepal, so setting it up again was the first thing that came into mind.<br>
it's an open-source, **self hosted dns hole** that acts like a bridge between internet and the devices connected to it.. it helps blocking ads, trackers and more. we can import custom trackers list like [(found on github)](https://github.com/topics/pihole-blocklists). it's also recommended to put it in anonymous mode so that none of the web page visited by me or my roommates are stored in the logs *(pi-hole settings > privacy)*.<br>

![pi-hole on its duty..]({{ "/assets/images/homelab-v1/pihole.png" | relative_url }}){: width="720" style="display: block; margin: 0 auto;"}
<p align="center"><i>pi-hole on its duty..</i></p>

after the pi-hole is configured, all you need to do is add the ip address of pi-hole as your dns while connecting to the wifi (or set it as primary dns on your switch or router), then voila, the pi-hole will start blocking all the trackers and ads.

##### [tailscale](https://tailscale.com/) - a cool vpn
if you tried accessing `drive.abiral.com` or `docs.abiral.com`, it might show page not available or some error message cause it would be a risky move to port forward the server and reveal my public ip to the world. i was originally thinking of configuring a **openvpn with wireguard**, but while researching about it, i found tailscale.
tailscale is a free tool on top of **wireguard** that helps me connect to my devices from anywhere in the world, suppose if you want to give access to drive to my family on the other side of the world, they can simply be added as a tailscale client. then they would be able to visit sites hosted in your local network and access the files without any issue. it didn't require any firewall configuration or complex vpn setup, it took only two commands per server to get it up and running.<br>
```bash
curl -fssl https://tailscale.com/install.sh | sh
sudo tailscale up
tailscale status  # check tailscale status
tailscale ip -4   # check your tailscale ip
```
tailscale gives seperate public ips to all the clients connected to a tailscale mesh. also in the tailscale dns settings overriding the dns nameserver to the tailscale ip of the pi-hole will help in filtering all vpn traffic through pihole. now, if i connect to my tailscale vpn on my phone's 5g network, it will serves everything via pihole without manually configuring the dns on each device.<br>

##### [nginx proxy manager](https://nginxproxymanager.com/)
**nginx (pronounced engine-ex)** basically acts like a middle man that diverts the traffic to multiple backend applications hosted on the network. it can also be used as **load balancer** or to handle **ssl/tls certificates**.<br>
nginx proxy manager basically is a **web-based management** for nginx that makes the nginx config much more simpler. suppose, if there is a service running at `192.168.0.11:3000`, we can configure `server1.example.com` to point to directly to the ip and port, which also helps to assign unique subdomain to each service hosted in same or different servers.<br>
the **nginx rules** helped in diverting the traffic to multiple backend applications that is hosted on the server and i used **wildcard subdomains** with cloudflare to automatically support any subdomains that we will add later in the rules.<br>

**wildcard subdomain with tailscale ip**
```
*.abiral.com. in a 100.83.xx.xx
```

##### installing [proxmox](https://www.proxmox.com/en/) - debian based ve platform
**proxmox** is the secret sauce to homelabbing. in simple terms it allows a device to be divided into multiple **virtual machines or containers**, each one having different operating system and its functionality. it can host two virtualization methods, one is **kvm (keneral-based virutal machine)**, it's a full blown operating system (like ubuntu or windows) that can be hosted in a ve, or **lxc (linux containers)**, which is typically a lightweight system container, meaning a container which behaves like a os with own system, ssh and more..<br>
after installing proxmox, its web based ui via `ip:8006`, helps to setup a new virtual environment in few clicks. just downloading latest **ubuntu server** or any linux iso and uploading it to the iso images.<br>

##### [jellyfin](https://jellyfin.org/) - a complete media system

for installation i quickly setup a directory with `docker-compose.yml` file inside it which had the necessary configurations. then quickly deploying the jellyfin server on port `8096` (used default port for jellyfin).<br>

since it only have **500 gb ssd** on the thinkcentre, there would be limitations on the numbers of movies or music i will upload to the server. we would need a whole seperate **nas (network attached storage)** server to host all the media contents to be served by jellyfin and adding a nas is a seperate addition for now we're skipping this step till the next upgrade. <br>

#### linux temperature monitoring using api, a fun project
it had been a while i played with a custom scripts to do some fun tasks, so i decided to create a **lightweight** temperature server that will track temperature of all the machines.<br>
i quickly created a **python script** that will act as a web server to receive temperature data from each servers and set it up on my raspberry pi, it's a simple script that listens to four values (server name, sensor name, temperature, and current time) and saves it in a jsonl file.<br>
on each server, i created a bash script called `log_temp.sh` which simply sends a curl post request to the temperature server with the required values, the script if based on the following command which helps to get temperature in each sensors.<br>
```bash
paste <(cat /sys/class/thermal/thermal_zone*/type) \
      <(cat /sys/class/thermal/thermal_zone*/temp) 2>/dev/null'
```
then i quickly added it as a cron job to run every 5 minutes.
```bash
chmod +x /home/jhilke-pi/scripts/linux-temp-logger/log_temp.sh  # executable permission

crontab -e   # open crontab
*/5 * * * * /home/hostname/scripts/linux-temp-logger/log_temp.sh >> /home/hostname/linux-temp-logger/temp_cron.log 2>&1
```

 <a href="https://github.com/itsabiral/linux-temp-log">github repo here.</a>
